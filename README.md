# bye-bye-yandex

С 29 июня 2026 Яндекс закрыл доступ по IMAP/SMTP для бесплатных
почтовых ящиков на собственных доменах — теперь для протоколов
требуется платная подписка Яндекс 360. Веб-интерфейс (`mail.yandex.ru`)
при этом продолжает работать бесплатно.

Этот проект — два независимых скрипта, которые вместе переносят
письма из Яндекс.Почты на любой другой IMAP-сервер **без включения
платного тарифа**:

1. **[`yandex-dump`](yandex-dump/)** — выгружает все письма из
   Яндекс.Почты в файлы `.eml`, используя внутренний веб-API вместо
   IMAP (тот же API, которым пользуется браузерный интерфейс).
2. **[`beget-upload`](beget-upload/)** — заливает эти `.eml` на новый
   сервер по обычному IMAP (`APPEND`), с сохранением папок, флага
   прочитанности и дат. Несмотря на название, работает с любым
   IMAP-сервером, не только Beget.

Между ними — обычные файлы `.eml` на диске, никакой магии и скрытого
состояния.

## Требования

- Python 3.8+ и модуль `venv` (в Debian/Ubuntu: `apt install python3-venv`).
- Внешняя зависимость одна — `requests`, и нужна она только
  `yandex-dump`; обёртка `dump.sh` при первом запуске сама создаст
  виртуальное окружение `.venv` и поставит её туда. `beget-upload`
  работает на чистой стандартной библиотеке.
- Как поставить окружение вручную (для запуска `yandex_dump.py` без
  обёртки) — см. [yandex-dump/README.md](yandex-dump/README.md#требования).

## Быстрый старт

```bash
git clone https://github.com/leksuss/bye-bye-yandex.git
cd bye-bye-yandex

cd yandex-dump
./dump.sh                          # подробно объяснит, что делать дальше

cd ../beget-upload
./upload.sh ../yandex-dump/dump box@ваш-новый-домен.ру
```

Подробные инструкции, все флаги и разбор проблем — в README каждого
подпроекта: [yandex-dump/README.md](yandex-dump/README.md),
[beget-upload/README.md](beget-upload/README.md).

## Важно про безопасность

`yandex-dump` **никогда не спрашивает и не видит ваш пароль от
Яндекса**. Вместо автоматического логина вы вручную копируете cookies
уже открытой в браузере сессии — это осознанное решение, а не
недоработка:

- так скрипт физически не может ни украсть, ни случайно залогировать
  пароль — его просто нет ни на одном этапе;
- так же честнее по отношению к антибот-защите Яндекса, чем
  автоматизированный логин с эмуляцией браузера.

Скрипт обращается **только** к `mail.yandex.ru` — это единственная
константа `BASE` в начале `yandex_dump.py`, можно проверить исходники
сами. Тем не менее файл с cookies по силе равен паролю (без второго
фактора) — не публикуйте его, не коммитьте в git, удаляйте после
использования.

`beget-upload` тоже не хранит и никуда не передаёт пароль от нового
ящика, кроме самого IMAP-сервера, к которому вы подключаетесь; ввод —
через `getpass` (не отображается на экране) или переменную окружения.

## Почему не через официальный API/протокол

Официального способа выгрузить письма без IMAP/SMTP и без оплаты
подписки Яндекс не предоставляет. Внутренний веб-API, который
использует `yandex-dump`, не документирован и может измениться в
любой момент — отсюда набор защитных механизмов в коде (автоопределение
структуры ответов, ретраи, подстраиваемые параметры) и подробный
раздел troubleshooting в README `yandex-dump`. Используйте на свой
страх и риск и не забывайте о разумных паузах между запросами, чтобы
не создавать нагрузку на чужой сервис.

## Структура репозитория

```
bye-bye-yandex/
├── yandex-dump/          # выгрузка писем из Яндекс.Почты
│   ├── yandex_dump.py
│   ├── dump.sh
│   └── README.md
├── beget-upload/          # заливка на новый IMAP-сервер

│   ├── upload.sh
│   └── README.md

```
