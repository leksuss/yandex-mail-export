# beget-upload

Заливает `.eml`-файлы, выгруженные скриптом [`yandex-dump`](../yandex-dump),
на новый почтовый сервер по IMAP (`APPEND`), с сохранением папок, флага
прочитанности и исходной даты писем.

Несмотря на название, скрипт работает с **любым сервером, поддерживающим
стандартный IMAP** — просто у Beget он написан по умолчанию. Чтобы
использовать другой сервер (свой Dovecot, mailcow, другой хостинг),
достаточно передать `--host` и `--port`.

Использует только стандартную библиотеку Python — ничего ставить не
нужно.

## Требования

- Python 3.8+
- Каталог с результатом `yandex-dump` (там должен лежать
  `manifest.jsonl`)
- Доступ по IMAP к целевому ящику (логин + пароль)

## Быстрый старт

```bash
cd beget-upload
./upload.sh /путь/к/dump box@ваш-домен.ру
```

`upload.sh` сначала всегда делает **dry-run** — покажет, какие папки
куда поедут и сколько писем в каждой, без единой реальной записи на
сервер — и только после вашего подтверждения запускает заливку.
Пароль запрашивается отдельно (`getpass`, не отображается на экране и
не пишется в историю шелла) — либо можно передать его через переменную
окружения `IMAP_PASS`.

Напрямую через Python:

```bash
python3 beget_upload.py --dump /путь/к/dump --user box@ваш-домен.ру --dry-run
python3 beget_upload.py --dump /путь/к/dump --user box@ваш-домен.ру
```

По умолчанию скрипт стучится на `imap.beget.com:993` — если у вас
другой хостинг или сервер, переопределите:

```bash
python3 beget_upload.py --dump /путь/к/dump --user box@example.com \
  --host imap.example.com --port 993
```

## Маппинг папок

Стандартные папки Яндекса переименовываются в стандартные IMAP-папки:

| Яндекс | IMAP |
|---|---|
| Входящие / Inbox | `INBOX` |
| Отправленные / Sent | `Sent` |
| Черновики / Drafts | `Drafts` |
| Спам / Spam | `Spam` |
| Удалённые / Trash | `Trash` |
| Архив / Archive | `Archive` |

Остальные (пользовательские) папки создаются на сервере с тем же
именем, включая вложенность и кириллицу (кодируется в IMAP UTF-7
автоматически). Переопределить маппинг конкретной папки:

```bash
python3 beget_upload.py --dump /путь/к/dump --user box@example.com \
  --folder-map 'Рассылки=Newsletters' --folder-map 'Заказы=INBOX.Заказы'
```

## Если в ящике на новом сервере уже есть письма

Например, если MX-записи домена уже переключены на новый сервер и туда
успела прийти часть свежей почты, пересекающейся с архивом — обычный
`APPEND` создаст дубликаты, IMAP это никак не проверяет сам.

Флаг `--dedupe` решает это: перед заливкой в каждую папку скрипт
разово вычитывает у сервера заголовки `Message-ID` уже лежащих там
писем (без скачивания тела, батчами — быстро даже на тысячах писем) и
пропускает совпадения:

```bash
python3 beget_upload.py --dump /путь/к/dump --user box@example.com --dedupe
```

Если ящик на новом сервере создан только что и точно пуст — флаг не
нужен, он просто немного замедлит старт.

## Резюмируемость

Список уже успешно залитых писем хранится в
`<dump>/uploaded_<логин>.txt`. Прервали — запустите ту же команду
снова, дольёт только недостающее.

## Все параметры

| Флаг | Что делает |
|---|---|
| `--dump DIR` | каталог с выгрузкой (обязателен, должен содержать `manifest.jsonl`) |
| `--user LOGIN` | логин ящика на новом сервере (обязателен) |
| `--host` | IMAP-хост (по умолчанию `imap.beget.com`) |
| `--port` | IMAP-порт (по умолчанию `993`, SSL) |
| `--dry-run` | только показать план по папкам, ничего не заливать |
| `--dedupe` | пропускать письма, чей `Message-ID` уже есть в целевой папке |
| `--folder-map "Откуда=Куда"` | переопределить назначение конкретной папки (можно несколько раз) |
| `--delay 0.05` | пауза между заливками, сек |

Пароль: переменная окружения `IMAP_PASS`, либо интерактивный запрос
при отсутствии переменной.

## Планы на будущее

Скрипт сознательно устроен максимально сервер-агностично (обычный
IMAP APPEND) — расширение под других провайдеров, у которых заливка
устроена иначе (не по IMAP, а через собственный API), потребует
отдельного скрипта в духе `<provider>-upload/`, но переиспользует всё,
что не зависит от протокола (чтение `manifest.jsonl`, маппинг папок,
дедупликация по `Message-ID`).
